## 1. 游戏核心循环

本作是一款 **放置（idle）+ 数值成长** 修仙游戏。核心循环如下：

```
打坐修炼 / 自动修炼 → 积累修为 → 突破境界（灵力倍率×1.2 复利）
        ↓
   提升"修炼层数"(等级) → 解锁更高秘境 / 更高装备境界要求
        ↓
   抽卡招募角色 → 组建宗门队伍（最多 5 人出战）
        ↓
   挂机探索 / 抽卡 / 炼丹 → 获取装备 · 灵宠 · 素材 · 灵石
        ↓
   宗门成员共享装备与灵宠 → 装备 Build + 灵宠出战 + 套装加成 → 战力飙升
        ↓
   挂机队伍协同战斗 + 小剧场增益 → 挑战更高秘境更高难度 → 更优掉落 → 反哺成长
```

**最重要的一条认知**：等级（修炼层数）本身**不直接增加**攻击/生命等战斗数值。真正让你变强的是 **角色阵容、装备 Build、灵宠、套装、丹药永久加成、转生**。等级的作用是：
- 抬高「修为上限」与「境界名称」；
- 每次突破让 **灵力获取倍率 ×1.2 复利叠加**（越往后灵力回得越快）；
- 解锁可装备的装备「境界要求」(`requiredRealm`)；
- 作为 Build 匹配度判定外的另一条进度主线。

## 2. 宗门系统：队伍与角色管理

游戏从「单角色」升级为「宗门」模式，底部导航的「宗门」即原「人物」页面（`Cultivation.vue`）。

### 2.1 宗门成员

- 抽卡获得的角色会加入宗门，**宗门上限人数** 由 `maxSectSize` 控制；
- 出战队伍最多 **5 人**，从宗门成员中编队；挂机时按队伍总 Build 强度结算战斗；
- **重复角色自动转换**：抽到已有的角色会自动转为「人精华」（3★=30、4★=60、5★=70），用于角色突破境界。
  ```
  essenceAmount = star × 10 + (star >= 4 ? 20 : 0)
  ```

### 2.2 宗门成员管理界面

在「宗门」页面可以：
- **下拉选择**查看任一已加入队伍的角色（避免一次展示过多）；
- 查看角色基础属性、定位、流派、天赋；
- **装备系统**：12 个装备槽位（武器/头部/衣服/裤子/鞋子/肩甲/手套/护腕/项链/戒指×2/腰带），手动穿脱或**一键自动穿戴**最优装备；
- **灵宠系统**：每个角色可装备 1 只灵宠，手动装备/卸下；
- **一键卸装**：快速卸下该角色所有装备与灵宠（归还共享背包）；
- 查看未出战的**候补成员**列表。

### 2.3 共享背包

宗门所有成员**共享同一个背包**：
- 装备、灵宠在背包中统一管理；
- 给某角色装备的物品会从背包移出，卸下后归还背包；
- 「一键自动穿戴」会从共享背包中为该角色挑选当前最优装备。

## 3. 角色定位系统（五大修仙定位）

每个角色拥有一个**定位（role）**，影响挂机战斗中的行动描写与小剧场表现。定位名称采用修仙风命名，但可直观对应传统 RPG 角色：

| 定位 | 修仙名 | 图标 | 对应传统定位 | 说明 |
| --- | --- | --- | --- | --- |
| vanguard | **先锋** | ⚔️ | 主C | 核心输出，队伍的主要战力担当 |
| blade | **刀锋** | 🗡️ | 附C | 副位输出，补充伤害与连击 |
| herb | **药引** | 🌿 | 辅助 | 辅助治疗，为队伍提供续航 |
| shield | **护法** | 🛡️ | 坦克 | 承伤坦克，保护队友免受致命打击 |
| tactician | **掌阵** | 📐 | 指挥官 | 战术核心，掌控全局节奏与阵法 |

### 3.1 角色命名规则

角色名按星级分风格：
- **3★**：武侠风命名（墨风、青石、烈无涯、冷月、惊鸿……）；
- **4★**：过渡风格，半武侠半仙侠（凌霜剑姬、玄机子、赤焰灵尊……）；
- **5★**：仙侠风命名（太虚剑帝、混元道祖、九阳焚天真君、万古冰帝……）。

### 3.2 挂机中的角色行动

挂机日志会**逐一描写每个队伍成员**的战斗行动（而非单纯的「你XXX」），按定位调用不同文案池：

```
先锋：挺身而出，以凌厉攻势直取敌人要害……
刀锋：身影如鬼魅般闪烁，快剑连斩三道……
药引：退后一步，指尖凝出碧绿灵光，为队友恢复气血……
护法：横身挡在队伍前方，护体真气化作金钟罩……
掌阵：在后方掐指推算，指点道：「左翼包抄，中军压上！」……
```

> 气血耗尽的角色会显示「气血耗尽，无法参战……」并跳过行动。

## 5. 修炼与突破：人物成长与修炼层数的关系

这是本手册最关键的一节，直接回答「人物成长和修炼层数的关系是什么？有哪些数据上面的成长」。

### 5.1 修炼层数 = 境界（共 126 层 / 14 大境 × 9 重）

`realm.js` 定义了 14 个大境界，每个 9 重，共 **126 级**：

```
练气 → 筑基 → 金丹 → 元婴 → 化神 → 返虚 → 合体 → 大乘
→ 渡劫 → 仙人 → 真仙 → 金仙 → 太乙 → 大罗
```

每一重的「修为上限 `maxCultivation`」随等级**线性阶梯式增长**。例如练气一重 100，练气九重 900，筑基一重 1000，……大罗九重 1.4 亿。

### 5.2 打坐修炼的两条指数曲线

```
单次消耗灵力 = floor( 8 × 1.25^(等级-1) )
单次获得修为 = floor( 3 × 1.28^(等级-1) )
```

因为 **修为增速(1.28) > 消耗增速(1.25)**，且修为上限只是线性增长，所以：
- **低等级**：需要几十次打坐才能填满一重；
- **高等级**：一次打坐获得的修为远超当前上限，会**自动连跨多阶突破**。

### 5.3 关键数值表（代表等级）

| 等级 | 境界 | 修为上限 | 单次耗灵力 | 单次修为 | 打坐次数(填满) | 突破成功率 | 被动修为/10秒* |
| ---: | --- | ---: | ---: | ---: | ---: | ---: | ---: |
| 1 | 练气一重 | 100 | 8 | 3 | 33.3 | 100% | 1 |
| 5 | 练气五重 | 500 | 19 | 8 | 62.5 | 94% | 4 |
| 9 | 练气九重 | 900 | 47 | 21 | 42.9 | 88% | 7 |
| 10 | 筑基一重 | 1,000 | 59 | 27 | 37.0 | 86.5% | 8 |
| 19 | 金丹一重 | 3,000 | 444 | 255 | 11.8 | 73% | 15 |
| 27 | 金丹九重 | 7,000 | 2,646 | 1,838 | 3.8 | 61% | 21 |
| 28 | 元婴一重 | 8,000 | 3,308 | 2,353 | 3.4 | 60% | 22 |
| 45 | 化神九重 | 35,000 | 146,936 | 156,444 | 0.22 | 60% | 36 |
| 63 | 合体九重 | 170,000 | 8,156,630 | 13,308,814 | 0.01 | 60% | 50 |
| 90 | 仙人九重 | 2,600,000 | 3,373,503,341 | 10,442,597,849 | 0.00 | 60% | 72 |

\* 被动修为为近似值，公式为 `max(1, floor(等级×0.8 + Build强度×0.00001))`，每 10 秒结算一次。

### 5.4 突破（tryBreakthrough）带来的数据成长

```
灵力     += 100 × 当前等级
灵力倍率  spiritRate ×= 1.2      ← 复利！每次突破灵力回得更快
境界名称  更新为下一重
修为      清零，修为上限改为下一重的 maxCultivation
突破次数  +1
```

**突破成功率**：
```
baseChance = max(0.6, 1 - (等级-1) × 0.015)
最终成功率  = min(1, baseChance + 渡厄丹加成)
```
- 1 级 100%，约 27 级后触底 **60%** 地板；
- 高境界频繁失败，建议用 **渡厄丹**（`breakthroughRate +10%`，可叠加至 +90%）兜底。

## 6. 转生渡劫系统

达到 **50 级** 可转生，重置等级为 1 但获得**永久属性加成**：

```
第 N 世转生加成：
  攻击 +5N    生命 +50N    防御 +3N    速度 +2N
  修炼效率 +10%N    灵力效率 +10%N
```
- 加成**永久沉淀**到 `baseAttributes` 与倍率，越转越强；
- 灵石、装备、物品、宗门成员**全部保留**；
- 适合中后期卡 Build 或想冲极限数值的玩家。

## 7. 装备与 Build 体系

### 7.1 装备品质（rarity）

| 品质 | 名称 | 颜色 | 评分倍率 | 词条数 |
| --- | --- | --- | --- | --- |
| common | 凡品 | 灰 | ×1 | 0 |
| uncommon | 良品 | 绿 | ×1.5 | 0~1 |
| rare | 上品 | 蓝 | ×2 | 1~2 |
| epic | 极品 | 紫 | ×3.5 | 2~3 |
| legendary | 仙品 | 金 | ×6 | 3~4 |
| mythic | 神品 | 红 | ×10 | 4~5 |

### 7.2 装备槽位（12 + 法宝）

```
weapon 武器 / head 头部 / body 衣服 / legs 裤子 / feet 鞋子
shoulder 肩甲 / hands 手套 / wrist 护腕 / necklace 项链
ring1 戒指 / ring2 戒指 / belt 腰带    ← 共 12 槽，宗门成员可装备
artifact 法宝                            ← 仅高难 Boss / 抽卡产出
```

### 7.3 词缀（affix）与品质倍率

- 词条按 `tier` 分档：`tier1 ×1`、`tier2 ×1.5`、`tier3 ×2.5`；
- 高 tier 词条（如 `增伤/减伤/战意/抗性`）对 Build 评分贡献更大；
- 单件装备评分公式：
```
装备评分 = round( (基础分 + 词条分 + 套装分) × 品质倍率 × (1 + 强化等级×0.1) )
```

### 7.4 套装（setBonuses）

集齐同套装多件触发百分比加成：

| 套装件数 | Build 加成分 |
| --- | --- |
| 2 件 | +200 |
| 3 件 | +300 |
| 4 件 | +500 |
| 5 件 | +800 |

游戏内置 10 套（烈焰刃/玄铁卫/疾风影/血牙/雷霆击/不灭体/反击道/灵气髓/战斗王/道仙尊），覆盖攻、防、速、吸血、暴击、生命、修炼等方向。

### 7.5 Build 强度（总评分）

```
Build强度 = Σ(每件已装备评分) + Σ(套装激活分)
```
- 挂机时，队伍总 Build = 所有出战成员的 Build 之和；
- 这是挂机「推荐 Build 匹配度」的判定基准。

### 7.6 Build 强度参考阶梯（BUILD_TIERS）

| 档位 | 名称 | 参考 Build | 含义 |
| --- | --- | --- | --- |
| 入门 | 入门 | 68,000 | 全稀有(+10)基础满装 |
| 强力 | 强力 | 686,000 | 传说/史诗混搭(+20)含套装 |
| 顶级 | 顶级 | 5,090,000 | 全神话(+50)含套装 |
| 极限 | 极限 | 20,000,000 | 全神话(+100)双五件套·混沌级 |

### 7.7 装备强化 / 洗练 / 出售

- **强化**：每级 `评分 ×(1+0.1×强化等级)`，吃强化石；可用**淬灵丹**提升下次成功率；
- **洗练**：重roll词条，吃洗练石；可用**定灵丹**保底不降属性；
- **出售**：`max(1, round(评分 × 0.1))` 灵石；
- **分解**：产出强化石/洗练石。

## 8. 灵宠系统与放生报恩

灵宠出战后为人物提供**全属性加成**（攻击/防御/生命 + 一众战斗属性）。加成由「品质 × 星级 × 等级 × 阶位」综合决定：

```
qualityBonusMap:  divine 15% / celestial 12% / mystic 9% / spiritual 6% / mortal 3%
每星加成:         各品质 1%~2%/星
每级加成:         (等级-1) × 基础加成 × 10%
每 5 星阶位:      阶位 × 基础加成 × 50%
最终加成 = 基础 + 星 + 等级 + 阶位（战斗属性取一半）
```

### 8.1 灵宠操作

- **升级**（`upgradePet`）：吃灵宠精华，按品质倍率提升战斗属性；
- **升星**（`evolvePet`）：消耗同名同品质灵宠，星级 +1，并返还被吃灵宠已耗精华；
- **装备到角色**（`equipCharacterPet`）：宗门成员可各装备 1 只灵宠，加成实时叠加；
- **出战/召回**：同一时刻每角色仅 1 只出战。

### 8.2 灵宠放生报恩

放生灵宠会根据其**稀有度**返还一定量的**灵宠精华**作为「报恩」：

```
返还精华 = 稀有度基础 + 等级×2 + 星级×5
```

| 灵宠品质 | 基础返还精华 |
| --- | ---: |
| 神品 divine | 100 |
| 仙品 celestial | 60 |
| 玄品 mystic | 35 |
| 灵品 spiritual | 20 |
| 凡品 mortal | 10 |

> 放生后灵宠感恩离去，精华实时到账。养成高星高等级的灵宠放生回报更丰厚。

## 9. 挂机探索系统（含小剧场）

挂机是放置收益的核心。每 **15 秒** 一场遭遇，按地图 `rewards` 掉落结算。

### 9.1 队伍协同战斗

- 挂机时按**队伍总 Build 强度**与地图推荐 Build 计算匹配度；
- 每个出战成员有**独立气血条**，气血耗尽则无法参战；
- 日志会**逐一描写每个角色的战斗行动**（按定位调用不同文案池）；
- **全队力竭**（所有成员气血归零）时挂机提前终止。

### 9.2 小剧场系统（idleBuffs）

挂机过程中会**随机触发队伍小剧场**，为本次挂机提供临时增益或减益 buff：

- 共 **15 种小剧场**，涵盖切磋、寻宝、内讧、感悟、中毒、灵泉等情境；
- buff 类型包括：`cultivation`（修炼）、`combat`（战斗）、`attack`（攻击）、`speed`（速度）、`luck`（气运）；
- buff 有**持续场次**，每场遭遇后递减，归零后失效；
- 部分为**负向 buff**（内讧、士气低落、受惊等），降低当次挂机效率。

**小剧场示例**：

| 小剧场 | buff 类型 | 效果 | 持续 | 性质 |

## 13. 战斗系统公式

战斗为**回合制**（最多 10 回合，超时判负），按速度决定先手。伤害结算顺序：

```
1) 基础伤害 = 攻击 × (1 + 战斗属性提升 combatBoost)
2) 暴击：最终暴击率 = 攻方暴击率×(1+combatBoost) − 守方抗暴×(1+抗性提升)
         触发 → 伤害 ×= (1.5 + 暴伤强化)
3) 连击：触发 → 伤害 ×= 1.3
4) 吸血：触发 → 回血 = 造成伤害 × 30%
5) 眩晕：触发 → 守方本回合无法反击
6) 最终增伤：伤害 ×= (1 + 最终增伤)
7) 防御减伤：伤害 ×= 100 / (100 + 有效防御)，有效防御 = 防御×(1+combatBoost)
8) 若本次为暴击 → 再 ×= (1 − 暴伤减免)
9) 最终减伤：伤害 ×= (1 − 最终减伤)
```
- **闪避**：实际闪避率 = 守方闪避率 − 攻方抗闪避；
- **反击**：守方反击率 − 攻方抗反击；
- **先手**：速度高者先攻。

> 实战要点：**最终增伤/减伤、暴伤、吸血、连击** 是后期碾压的核心；堆叠 `combatBoost` 与 `resistanceBoost` 能同时放大攻防两端。

## 14. 进阶策略与成长路线

### 前期（练气 ~ 金丹，1~27 级）
- 主线：手动/自动打坐把修为堆上去，靠突破把 `spiritRate` 复利拉起来；
- 资源：低难挂机（青萝林/迷雾谷 游历~试炼）攒灵石、灵草；
- 宗门：用综合池/人物池抽角色组队，5 人满编；
- 炼丹：优先 **聚灵丹(灵力)、聚气丹(修炼)、洗髓/锻骨丹(永久属性)**；
- 装备：抽卡综合池拿第一套良品/上品，凑 2 件套即可。

### 中期（元婴 ~ 化神，28~45 级）
- 目标：队伍总 Build 冲到 **入门(6.8万)~强力(68.6万)**；
- 装备：进凤凰窟/龙渊 凶险~绝境，挂机出极品/仙品；集齐 1~2 套 4 件套；
- 灵宠：玄品/仙品出战，开始升级+升星；多余灵宠**放生报恩**换精华；
- 突破：60% 地板后用 **渡厄丹** 兜底；
- 丹药：悟道丹+寻宝丹叠加，挂机收益最大化；
- 重复角色转人精华用于突破。

### 后期（返虚 ~ 大乘，46~72 级）
- 目标：Build 冲 **顶级(509万)**，挑战鬼荒原/冰雪宫 绝境~灭世；
- 装备：神品为主，法宝(artifact)优先补位；强化 +20~+50；
- 灵宠：仙品/神品满星，精华集中养 1~2 只核心；
- 转生：满 50 级可转生，永久 +攻/血/防/速 与 10% 效率，越转越强。

### 终局（渡劫 ~ 大罗，73~126 级）
- 目标：Build 逼近 **极限(2000万)**，混沌界 灭世稳定挂机；
- 全神话双五件套、法宝满配、神品满星灵宠、多次转生叠永久属性；
- 利用被动修为与离线结算，放置即可持续逼近大罗九重 1.4 亿修为上限。

### 通用技巧
1. **队伍编成**：5 人满编，注意定位搭配（先锋+刀锋输出、护法抗伤、药引续航、掌阵增益）；
2. **Build 优先于等级**：与其猛冲等级，不如先把 Build 堆到匹配当前图；
3. **套装 > 散件**：4~5 件套的 Build 分与百分比加成远超单件；
4. **小剧场 buff**：挂机时留意小剧场触发，正向 buff 可显著提升效率，负向 buff 时可考虑暂停或换图；
5. **灵力倍率复利**是隐藏引擎：多突破 = 灵力回得快 = 修炼不停；
6. **永久丹药**早吃早享受，转生不丢；
7. **灵宠放生**：养成后再放生回报更高（等级×2 + 星级×5）；
8. **匹配度 < 100% 别硬挂**：会力竭提前结束，先把 Build 或难度降一档。

## 15. 数据附录：关键公式汇总

```js
// —— 修炼 ——
单次消耗灵力 = floor(8 × 1.25^(等级-1))
单次获得修为 = floor(3 × 1.28^(等级-1))
突破成功率   = min(1, max(0.6, 1 - (等级-1)×0.015) + 渡厄丹)
突破奖励     = 灵力 +100×等级;  spiritRate ×= 1.2
被动修为/10s = max(1, floor(等级×0.8 + Build×0.00001))
修为上限     = realm.js 线性阶梯（100 → 1.4亿）

// —— 宗门 ——
重复角色转化 = star × 10 + (star >= 4 ? 20 : 0)   // 3★=30 / 4★=60 / 5★=70
队伍总Build  = Σ(各出战成员Build)

// —— 装备评分 ——
装备评分 = round((基础分 + 词条分 + 套装分) × 品质倍率 × (1 + 强化等级×0.1))
品质倍率   = common1 / uncommon1.5 / rare2 / epic3.5 / legendary6 / mythic10
词条tier倍 = tier1×1 / tier2×1.5 / tier3×2.5
套装激活分 = 2件+200 / 3件+300 / 4件+500 / 5件+800
Build强度  = Σ(已装备评分) + Σ(套装激活分)

// —— 挂机 ——
推荐Build(难度) = ZONE_BUILD_BASE × 难度缩放(0.3/0.6/1.0/1.6/2.5)
匹配度         = 队伍总Build ÷ 推荐Build
胜利修为       = round(5 × 地图难度 × (1 + 匹配度×0.5))

// —— 小剧场 buff ——
buff持续 = duration 场遭遇，每场 -1，归零失效
增益类型 = cultivation(修炼) / combat(战斗) / attack(攻击) / speed(速度) / luck(气运)

// —— 灵宠加成 ——
最终加成 = 品质基础(3%~15%) + 星级(1%~2%/星) + 等级(基础×10%×(级-1)) + 阶位(基础×50%×floor(星/5))
战斗属性取最终加成的一半

// —— 灵宠放生报恩 ——
返还精华 = 稀有度基础(凡10/灵20/玄35/仙60/神100) + 等级×2 + 星级×5

// —— 抽卡 ——
角色概率   = 综合池15% / 人物池25%（其余为装备/灵宠/资源）
角色星级   = 5星3% / 4星20% / 3星77%
保底       = 人物池/综合池 每5次保底4★+
资源灵石   = 20~80（低于单次抽卡成本）

// —— 战斗 ——
伤害 = 攻击×(1+combatBoost) ×[暴击1.5+暴伤] ×[连击1.3] ×(1+最终增伤)
     × 100/(100+防御×(1+combatBoost)) ×(1−暴伤减免,若暴击) ×(1−最终减伤)
闪避 = 守方闪避 − 攻方抗闪避；  反击 = 守方反击 − 攻方抗反击
```
